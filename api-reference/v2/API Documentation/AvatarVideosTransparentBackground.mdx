## Overview

Create avatar videos with transparent backgrounds (alpha channel) for easy compositing over other content. Perfect for presentations, video editing, or web applications.

<Info>
**Specialized Scenario:** This guide focuses on transparent background videos. For the complete avatar video creation guide, see [Create Avatar Videos](/api-reference/v2/API Documentation/CreateAvatarVideos).
</Info>

---

## When to Use Transparent Backgrounds

**Best for:**
- Overlaying avatars on presentations
- Adding avatars to existing videos  
- Website video backgrounds
- Video editing projects
- Green screen replacements
- AR/VR applications

---

## Quick Example

Create a video with transparent background:

```bash
curl --request POST \
  --url 'https://api.jogg.ai/open/v2/create_video_from_avatar' \
  --header 'x-api-key: YOUR_API_KEY' \
  --header 'Content-Type: application/json' \
  --data '{
    "avatar_id": 81,
    "avatar_type": 0,
    "voice_id": "en-US-ChristopherNeural",
    "script": "Welcome! This avatar has a transparent background.",
    "aspect_ratio": "portrait",
    "screen_style": 2,
    "caption": false
  }'
```

**Response:**

```json
{
  "code": 0,
  "msg": "Success",
  "data": {
    "video_id": "video_123456"
  }
}
```

<Check>
Set `screen_style: 2` for green screen or `screen_style: 3` for WebM with alpha channel
</Check>

---

## Key Parameters

| Parameter | Value | Description |
|-----------|-------|-------------|
| `screen_style` | `2` | Green screen background (chroma key) |
| `screen_style` | `3` | WebM with alpha channel (true transparency) |
| `avatar_id` | required | Choose full-body avatar for best results |
| `aspect_ratio` | required | portrait/landscape/square |

**Output Formats:**
- **Green Screen (screen_style: 2)**: MP4 with green background - use chroma key in video editor
- **WebM (screen_style: 3)**: WebM with alpha channel - true transparency, no keying needed

## Comparison

| Feature | Green Screen (2) | WebM Alpha (3) |
|---------|------------------|----------------|
| **Format** | MP4 | WebM |
| **Transparency** | Chroma key required | Native alpha channel |
| **Compatibility** | All editors | Modern browsers/editors |
| **File Size** | Smaller | Larger |
| **Quality** | Good (may have green spill) | Excellent (clean edges) |
| **Best For** | Video editing | Web applications |

## Best Practices

<Tip>
**Quality Tips:**
- Use full-body avatars for best results
- Test composite in target environment
- For web, use WebM with alpha (screen_style: 3)
- For video editing, use green screen (screen_style: 2)
- Avoid captions if compositing over text-heavy backgrounds
</Tip>

**Lighting Considerations:**
- Green screen works best with even lighting
- WebM alpha has cleaner edges
- Test with your background before bulk production

## Troubleshooting

<AccordionGroup>
  <Accordion title="Green spill on edges">
    **Issue:** Green color bleeding onto avatar
    
    **Solution:**
    - Use "Spill Suppression" in video editor
    - Increase key tolerance settings
    - Consider using WebM with alpha instead
  </Accordion>
  
  <Accordion title="WebM not supported">
    **Issue:** Browser or editor doesn't support WebM
    
    **Solution:**
    - Use green screen format (screen_style: 2)
    - Convert WebM to other format with alpha preservation
    - Check browser/editor compatibility
  </Accordion>
  
  <Accordion title="Rough edges">
    **Issue:** Avatar edges look jagged
    
    **Solution:**
    - Adjust edge feathering in video editor
    - Use higher quality avatar
    - Increase key softness settings
  </Accordion>
</AccordionGroup>
